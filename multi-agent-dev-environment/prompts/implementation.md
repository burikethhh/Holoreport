# Implementation Prompts for Multi-Agent Development Environment

## Overview
This document provides prompts and guidelines for the implementation phase of the multi-agent development environment project. It is designed to assist the Full Stack Leader and Assistant in executing development tasks effectively.

## Prompts

### Task Breakdown
1. **Define Development Tasks**: 
   - What are the key features to be implemented in this phase?
   - Break down each feature into smaller, manageable tasks.

2. **Assign Responsibilities**:
   - Which tasks will be handled by the Full Stack Leader?
   - Which tasks will be delegated to the Assistant?

### Development Process
3. **Code Structure**:
   - What is the proposed structure for the codebase?
   - How will modules and components be organized?

4. **Integration Points**:
   - Identify integration points between different components.
   - How will the orchestrator facilitate communication between agents?

### Testing and Validation
5. **Testing Strategy**:
   - What testing frameworks will be used?
   - How will unit tests and integration tests be structured?

6. **Validation Criteria**:
   - What criteria will be used to determine if a feature is complete?
   - How will feedback be gathered and incorporated?

### Documentation
7. **Documentation Requirements**:
   - What documentation is necessary for each feature?
   - How will the documentation be maintained throughout the development process?

### Review and Iteration
8. **Code Review Process**:
   - What will be the process for code reviews?
   - How will feedback be provided and addressed?

9. **Iteration Plan**:
   - How will the team handle iterations and updates based on testing and feedback?
   - What is the timeline for each iteration?

## Conclusion
Utilize these prompts to guide the implementation phase, ensuring a structured and efficient development process. Collaboration between the Full Stack Leader and Assistant is crucial for the success of the project.
